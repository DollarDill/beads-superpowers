# Design: Example Workflow Overhaul — yegge + jesse Agents + Karpathy CLAUDE.md

> **Date:** 2026-05-02
> **Epic:** beads-superpowers-dvl
> **Brainstorming bead:** beads-superpowers-dvl.10
> **Design beads:** beads-superpowers-dvl.3 (yegge agent), beads-superpowers-dvl.5 (CLAUDE.md template)
> **Status:** Approved

## Overview

Restructure `example-workflow/` from a single monolithic CLAUDE.md into a three-layer system: a slim Karpathy-style CLAUDE.md (behavioral guidelines + bd init beads section), a `yegge` orchestrator agent (complete FSM behavioral playbook), and a renamed `jesse` researcher agent. Install.sh gains agent installation to `~/.claude/agents/`.

## Architecture

### Three-Layer System

| Layer | File | Content | When Loaded | Lines |
|-------|------|---------|-------------|-------|
| Behavioral rules | `CLAUDE.md` | Verbatim Karpathy 4-principle file + bd init beads section | Session start (auto) | ~90 |
| Orchestration | `yegge.md` agent | FSM, triage, planning, rules, session protocol, output format | Session start (agent) | ~300 |
| Research | `jesse.md` agent | Deep research specialist, LSP-first, cross-reference | On dispatch by orchestrator | ~65 |

**Naming convention:** Agents named after project creators — Steve Yegge (beads) and Jesse Vincent (superpowers).

### File Layout

```text
example-workflow/
├── CLAUDE.md                    # Generic Karpathy-style project template
│                                # Verbatim forrestchang/andrej-karpathy-skills content
│                                # + bd init generated beads integration section
│
├── agents/
│   ├── yegge.md                 # Complete orchestrator agent (renamed from rpi-developer)
│   └── jesse.md                 # Deep research specialist (renamed from researcher.md)
│
└── README.md                    # Updated setup instructions
```

### Relationships

- `yegge.md` is self-contained — ALL orchestration logic lives here
- `CLAUDE.md` is project context — Claude Code auto-loads it for all agents
- `jesse.md` is dispatched by yegge at FSM state S2
- `install.sh` copies `yegge.md` + `jesse.md` to `~/.claude/agents/` (globally available)
- Users manually copy/adapt `CLAUDE.md` template into their projects

## Component Specifications

### 1. CLAUDE.md Template

**Composition:**

1. **Attribution header** — HTML comment crediting forrestchang/andrej-karpathy-skills (MIT)
2. **Verbatim Karpathy content** (~65 lines) — Fetched directly from `https://github.com/forrestchang/andrej-karpathy-skills/blob/main/CLAUDE.md`. The 4 principles:
   - Think Before Coding
   - Simplicity First
   - Surgical Changes
   - Goal-Driven Execution
3. **Beads integration section** — Generated by running `bd init` in a temp project, captured with `<!-- BEGIN BEADS INTEGRATION -->` / `<!-- END BEADS INTEGRATION -->` markers. NOT hand-written.

**Build process during implementation:**

```bash
# 1. Fetch Karpathy CLAUDE.md
curl -fsSL https://raw.githubusercontent.com/forrestchang/andrej-karpathy-skills/main/CLAUDE.md > /tmp/karpathy.md

# 2. Run bd init in temp dir, capture generated beads section from AGENTS.md
mkdir -p /tmp/beads-test && cd /tmp/beads-test && git init && bd init
# Extract content between BEGIN/END BEADS INTEGRATION markers from AGENTS.md

# 3. Combine: attribution + karpathy + beads section
```

**Design principles:**
- Works for ANY project, even without beads-superpowers
- Karpathy content is verbatim — no modifications, no paraphrasing
- Beads section is machine-generated — stays in sync with upstream bd CLI
- Users add project-specific content (architecture, gotchas, design decisions) between the two sections
- The original Karpathy header "Merge with project-specific instructions as needed" guides users

## Decisions Made

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Agent location | `example-workflow/agents/` installed to `~/.claude/agents/` | Clean separation; follows researcher.md pattern; global availability |
| Responsibility split | Yegge owns all orchestration; CLAUDE.md is slim project template | Keeps agent self-contained; CLAUDE.md stays generic |
| CLAUDE.md content | Verbatim Karpathy file + bd init beads section | No hand-written content; stays in sync with upstream sources |
| Agent naming | yegge (orchestrator) + jesse (researcher) | Honors both project creators: Steve Yegge (beads) + Jesse Vincent (superpowers) |
| Install method | install.sh copies agents to ~/.claude/agents/ globally | Consistent with existing skill/hook installation pattern |
| CLAUDE.md distribution | Manual copy by user | Project-level file shouldn't be auto-installed globally |

## Out of Scope

- Modifying `bd setup claude` upstream (depends on Issue #2140)
- Adding new skills — this is a restructure of existing content
- Changing the FSM states or skill invocations — content is reorganized, not redesigned
- Plugin-level agent definitions (yegge/jesse are user-level, code-reviewer is plugin-level)

## Risks

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| Karpathy CLAUDE.md URL changes or repo deleted | Low | Medium | Pin to specific commit SHA; include verbatim copy in our repo |
| `bd init` beads section format changes across bd versions | Medium | Low | Markers make the section replaceable; document bd version used |
| Users confused by three-layer system | Medium | Low | README explains setup; install.sh handles agents automatically |
| `jesse` name collision with user's existing agents | Low | Low | Unique enough; install.sh uses `cp -f` to overwrite |
