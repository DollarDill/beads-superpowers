# ADR-0003: Example Workflow Three-Layer Architecture (yegge + jesse + Karpathy CLAUDE.md)

**Date:** 2026-05-02
**Status:** Accepted
**Deciders:** Dillon Frawley

## Context

The `example-workflow/` directory provides a ready-to-copy development workflow for beads-superpowers users. The current design has a single monolithic `CLAUDE.md` (153 lines) that mixes project documentation with orchestration logic (FSM, triage, planning principles, critical rules, session protocol). A companion `researcher.md` agent exists but references a stale `implementer.md` that was already removed.

Problems:

1. **Monolithic CLAUDE.md.** Behavioral orchestration rules (the FSM) and project documentation are mixed in one file. Users who want the FSM must take the whole file; users who want project docs must read past the FSM.
2. **No behavioral guardrails.** The CLAUDE.md doesn't include proven LLM behavioral guidelines (like Karpathy's 4 principles for preventing overcomplication, assumption-making, and scope creep).
3. **Generic agent names.** `researcher.md` is descriptive but bland. No credit to project creators.
4. **No global agent installation.** Users must manually copy agents into each project's `.claude/agents/` directory.

Research findings (beads dvl.1 and dvl.2) revealed:
- Karpathy's actual CLAUDE.md style is architectural documentation, but the viral forrestchang file (106k stars, MIT licensed) provides proven behavioral guardrails
- `bd init` generates a beads integration section with `<!-- BEGIN/END BEADS INTEGRATION -->` markers for AGENTS.md, but does NOT generate CLAUDE.md content (upstream Issue #2140)

## Decision

Restructure `example-workflow/` into a three-layer system:

1. **CLAUDE.md** — Slim generic project template (~90 lines): verbatim copy of the forrestchang/andrej-karpathy-skills CLAUDE.md (4 behavioral principles) + beads integration section generated by `bd init`. No hand-written beads content.

2. **yegge.md** — Complete orchestrator agent (~300 lines): all FSM states, request triage, planning principles, critical rules, session protocol, output format. Named after Steve Yegge (beads creator). Self-contained — does not depend on CLAUDE.md for orchestration logic.

3. **jesse.md** — Research specialist agent (~65 lines): renamed from `researcher.md`. Named after Jesse Vincent (superpowers creator). Content unchanged.

Distribution: `install.sh` copies `yegge.md` and `jesse.md` to `~/.claude/agents/` for global availability. CLAUDE.md template is manually copied by users into their projects.

## Rationale

- **Separation of concerns.** CLAUDE.md is project context (loaded for all agents). The yegge agent is orchestration behavior (loaded for the primary session). Mixing them creates a file that's too long and serves two masters.
- **Verbatim Karpathy content.** The forrestchang file is MIT licensed, battle-tested (106k stars), and explicitly designed to be merged with project-specific instructions. Writing our own version would be inferior and drift from the community standard.
- **bd init for beads section.** Using the actual `bd init` output means the beads section stays in sync with the bd CLI. Hand-writing it would create a maintenance burden and drift risk.
- **Creator naming.** Memorable, short, and pays homage. `yegge` (orchestrator) and `jesse` (researcher) are immediately distinguishable and give the project personality.
- **Global agent installation.** Consistent with existing install.sh pattern (skills and hooks are already installed globally). Agents should be too.

## Consequences

**Positive:**
- Users get proven Karpathy behavioral guardrails out of the box
- CLAUDE.md works for ANY project, even without beads-superpowers
- Agent names are memorable and credit the open-source creators
- Three focused files are easier to understand and modify than one monolithic file

**Negative:**
- Breaking change: `researcher.md` → `jesse.md` requires users to update references
- Three-file setup is slightly more complex than single CLAUDE.md
- `bd init` beads section format may change across bd versions (mitigated by markers)

**Neutral:**
- `rpi-developer` name is retired; existing users need to switch to `yegge`
- CLAUDE.md template no longer contains beads commands inline — bd prime provides them dynamically
